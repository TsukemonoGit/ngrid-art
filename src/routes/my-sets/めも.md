# My Sets 機能 構想メモ

## ルート構成

```
/                            ← グリッドアート編集（Home、既存）
/emoji-sets                  ← みんなの絵文字セット閲覧（既存）
/my-sets                     ← 自分の30030一覧・作成・削除（新規）
/my-sets/[identifier]        ← 特定30030の編集（新規）
```

---

## ナビゲーション UI

`PageChange.svelte` のトグル方式を、3リンク並びのナビバーに変更する。

**ヘッダー：**

```
[Nostr Grid Art]   [Home] [Emoji Sets] [My Sets]   [Settings]
```

- 現在のページ: アクティブスタイル（塗りつぶし）
- その他のページ: アウトラインボタン
- スマホ: アイコンのみ表示（`isMobile` 判定を使う）

---

## /my-sets 一覧ページ

- ログインしていない場合はログインを促す
- 自分の pubkey で kind:30030 を取得して一覧表示
- 右上に「＋ 新規作成」ボタン
  - 作成後 `/my-sets/[identifier]` に遷移
- 各セットカードに「削除」ボタン
  - 削除前に**確認ダイアログ**を表示
  - 削除 = kind:5 を投稿。`e` タグ（対象イベントの id）と `a` タグ（`30030:<pubkey>:<identifier>`）の両方を含める（`a` タグ削除に対応していないリレーへの対処）

### 新規作成の挙動

- 作成ダイアログで以下を入力：
  - **タイトル**（必須）
  - **identifier**（dtag）：unixtime を初期値として自動入力。`[a-zA-Z0-9_-]` の範囲でユーザーが編集可能
- 既存の identifier と重複した場合は作成不可（重複チェックは取得済みの自分の kind:30030 一覧と突き合わせ）
- identifier は**新規作成ダイアログの時点でのみ**変更可能。編集ページへ遷移後は変更不可

**UI イメージ：**

```
My Emoji Sets                          [+ 新規作成]
──────────────────────────────────────────────────
[セット名A]  identifier: xxx  [編集]  [削除]
[セット名B]  identifier: yyy  [編集]  [削除]
```

## /my-sets/[identifier] 編集ページ

- URL の `identifier` でそのセットを特定
- ページ上部に「← My Sets」の戻るリンク

### セット情報の編集

- `title` タグ（セット名）の編集フォーム
- identifier（dtag）は URL から取得（変更不可）
- タイトルは編集可能

### 絵文字の追加

- **URL 直接入力** または **ファイルアップロード** の両方に対応
- ファイルアップロード時はサーバーを選択するドロップダウンを横に配置
  - 方式：**NIP-96** または **Blossom**
  - サーバー URL をドロップダウンで選択（画像のUI参照）
  - 選択したサーバー設定は**ローカルストレージに保存**し、次回起動時に復元
  - アップロード中はローディング表示
- shortcode 入力フォーム
- URL 入力 or アップロード後、その画像を**プレビュー表示**
- `["emoji", shortcode, url]` タグとしてセットに追加
- 同一リスト内でのショートコード被りがないように制御

### 現在の絵文字リスト

- セット内の絵文字を一覧表示
- **並び替え対応**（ドラッグ＆ドロップ等）
- 各絵文字に **削除ボタン** と **修正ボタン**
  - 削除前に**確認ダイアログ**を表示
  - 修正ボタンを押すとインライン編集フォームに展開
  - `shortcode: [____]  URL: [____]  [保存] [キャンセル]`
  - URL 欄の変更に合わせてリアルタイムで**画像プレビュー表示**

### 保存

- 「保存して公開」ボタン → kind:30030 として署名・投稿

---

## 実装順の目安

1. `PageChange.svelte` をナビバーに変更
2. `/my-sets` 一覧ページ（取得・表示・削除・新規作成導線）
3. `/my-sets/[identifier]` 編集ページ（セット編集・保存）
4. 絵文字の並び替え
5. 画像アップロード（NIP-96 / Blossom、サーバー選択＋ローカルストレージ保存）
