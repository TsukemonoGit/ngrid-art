# Nostr grid art 設計書

## 概要

Nostr のカスタム絵文字を1セル1絵文字のグリッドに配置し、
カスタム絵文字アートを作成・投稿するツール。
出力は Nostr kind 1 投稿として使用できるテキスト形式。

---

## 技術スタック

- **フレームワーク**：SvelteKit
- **Nostr ライブラリ（リレー通信）**：rx-nostr
- **Nostr ライブラリ（ログイン・署名）**：@konemono/nostr-login
- **対象プラットフォーム**：Web（PC / スマホ）

---

- グリッドの上下左右の端に「＋」ボタン（行・列を追加）

- スマホでは「−」ボタンはタップで表示、PCではホバーで表示

## グリッドの仕様

- 1セル = カスタム絵文字1個（正方形想定）
- 空セルは null 絵文字として出力する（後述）
- グリッドは行列構造で管理する
- グリッドの初期サイズ：8×8
- グリッドサイズの上限：縦64×横64

```typescript
type EmojiTag = ["emoji", string, string, string?];
// ["emoji", resolvedShortcode, url, "30030:pubhex:identifier"]
// ノラ絵文字は4要素目なし

type Cell = EmojiTag | null;
type Grid = Cell[][];
```

## 操作モデル

### 絵文字の選択状態

| 操作                                               | 挙動                 |
| -------------------------------------------------- | -------------------- |
| パレットで絵文字をクリック（選択していない絵文字） | 選択絵文字を切り替え |
| パレットで絵文字をクリック（すでに選択中の絵文字） | 選択解除             |
| 「選択解除」ボタン                                 | 選択解除             |

### セルのタップ挙動

| 状態         | セルの状態             | 挙動                     |
| ------------ | ---------------------- | ------------------------ |
| 絵文字選択中 | 空セル                 | 配置                     |
| 絵文字選択中 | 配置済み（別の絵文字） | 上書き                   |
| 絵文字選択中 | 配置済み（同じ絵文字） | コンテキストメニュー表示 |
| 選択解除中   | 配置済み               | コンテキストメニュー表示 |
| 選択解除中   | 空セル                 | 何もしない               |

### グリッドのスクロール

- WASDキーで2セル分スクロールする（PC・スマホ共通）
- ブラウザ標準のスクロール（マウスホイール・タッチスクロール）も有効
- スマホでは画面上にキーボード呼び出しボタン（⌨）を配置し、ソフトキーボードを呼び出せるようにする

### コンテキストメニュー項目

- **削除**：セルを null に戻す
- **パレットにコピー**：その絵文字をパレットの選択状態にする

## null 絵文字の仕様

カスタム絵文字はクライアント間で表示幅が統一されるため、
AA的な幅の一貫性を保つために空セルもカスタム絵文字として出力する。

### デフォルト null 絵文字

- アプリ固有の正方形・透明絵文字を用意する
- この絵文字はアプリ運営側が kind `30030` イベントとしてホストする
- ユーザーはこのアプリ用 30030 を自分のカスタム絵文字リストに登録しておくことで、
  コピペ先クライアントでも補完が効く
- null 絵文字はパレットとは独立して管理する（ユーザーの 10030 に未登録でも使用できる）

### null 絵文字の変更

- ユーザーはパレットから任意の絵文字を null 絵文字として選択できる
- 選択した絵文字が空セルの出力に使われる（背景として機能する）
- 全角スペース（`\u3000`）もオプションとして選択可能だが、
  クライアント依存で幅がズレる可能性があるため非推奨として明示する

## パレットの絵文字データ管理

### 取得フロー

1. ハードコードした既知リレー群からユーザーの kind `10002` を取得する
2. `10002` の読みリレーから kind `10030` を取得する
3. `10030` の `a` タグが指す kind `30030` を読みリレーから取得する（メイン）
4. 見つからない場合、`10030` の `a` タグのリレーヒントから `30030` を取得する（フォールバック）
5. `10030` の `emoji` タグ（ノラ絵文字）も列挙する

### 同名ショートコードの衝突解消

複数の `30030` に同名ショートコードが存在する場合、パレット読み込み時に以下のルールで解消する：

- `30030` を `pubkey+dtag` でソートし、先勝ち
- 後続の同名ショートコードには連番 suffix を付与する（例：`cat_2`、`cat_3`）
- 解消済みショートコードはパレットデータとともに確定し、localStorage に保存する

### ノラ絵文字

- `10030` に直接 `emoji` タグで登録されている絵文字
- `EmojiTag` の4要素目（30030参照）なしで管理する

### localStorage への保存

以下をlocalStorageに自動保存する：

- グリッドの内容（`Cell[][]`、EmojiタグをCellに含む）
- 衝突解消済みパレットの絵文字リスト
- null 絵文字の設定

起動時はlocalStorageから復元し、オフラインでも使用できる。
リレーからデータを受信した場合はパレットの絵文字リストを上書きする。

## 出力形式

グリッドを以下のルールで kind 1 の content テキストに変換する。

### 出力前トリム処理

出力テキストの生成前に、グリッドの上端・下端・左端・右端から
null のみで構成される行・列を除去する。
グリッドの表示・編集状態には影響しない。
トリム後のサイズ（例：`5列×3行`）をコピー・投稿エリアに表示する。

### 変換ルール

- 各セル：絵文字は `:shortcode:` 形式
- 空セル：設定中の null 絵文字の `:shortcode:` 形式
- 各行：セルをそのまま横に並べる
- 行間：改行（`\n`）で区切る

### 例

null 絵文字: `:blank:` / グリッド：

```
[null] [cat  ] [null]
[star ] [null ] [party]
```

出力 content：

```
:blank::cat::blank:
:star::blank::party:
```

### tags の出力

kind 1 イベントの `tags` には、使用した絵文字 + null 絵文字を重複なく列挙する。
30030参照あり（通常の絵文字）：

```json
["emoji", "cat", "https://example.com/cat.png", "30030:<pubhex>:<identifier>"]
```

30030参照なし（ノラ絵文字）：

```json
["emoji", "cat", "https://example.com/cat.png"]
```

## 機能一覧

### 1. ログイン

- `@konemono/nostr-login` を使用する
- v1はログイン済み・kind `10030` 所持を前提とする
- `10030` 未所持の場合はエラーメッセージを表示する

### 2. 絵文字選択・解除

- パレットから絵文字をクリックして選択状態にする
- 選択中の絵文字はプレビューエリアに表示される
- 選択解除ボタン、またはパレットで同じ絵文字を再クリックすると解除される

### 3. セルへの配置・上書き

- 絵文字選択中に空セルをタップすると配置される
- 絵文字選択中に配置済みセル（別の絵文字）をタップすると上書きされる

### 4. コンテキストメニュー

以下の条件でコンテキストメニューを表示する：

- 絵文字選択中に配置済みセル（同じ絵文字）をタップ
- 選択解除中に配置済みセルをタップ

メニュー項目：

- **削除**：セルを null に戻す
- **パレットにコピー**：その絵文字をパレットの選択状態にする

### 5. グリッドのスクロール

- WASDキーで2セル分スクロールする（PC・スマホ共通）
- ブラウザ標準のスクロール（マウスホイール・タッチスクロール）も有効
- スマホでは画面上にキーボード呼び出しボタン（⌨）を配置する

### 6. グリッドサイズ変更

- グリッドの上端・下端・左端・右端に `+` ボタンを配置する
- 下端の `+`：末尾に1行を追加する
- 右端の `+`：末尾に1列を追加する
- 上端の `+`：先頭に1行を挿入する
- 左端の `+`：先頭に1列を挿入する
- 行・列の削除ボタンも配置する
  - PC：ホバー時に削除ボタン（`−`）を表示
  - スマホ：グリッド端に常時表示、タップで確認ダイアログ→削除
- 配置済み絵文字が存在する行・列を削除しようとした場合は確認ダイアログを表示する

### 7. パレット絵文字の取得

「パレットの絵文字データ管理」セクションを参照。

### 8. 出力

- **contentをコピーボタン**：生成した content テキストをクリップボードにコピーする
- **kind 1で投稿ボタン**：`@konemono/nostr-login` で署名し、kind 1 イベントを `10002` の書きリレーに投稿する
- コピー・投稿エリアにトリム後のサイズを表示する
